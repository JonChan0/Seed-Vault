# LLM Frontends

Seed Vault works with **Claude Code** or **Antigravity CLI** (`agy`) as the LLM frontend. Both use the same skills, Python engines, and wiki format.

## Comparison

| | Claude Code | Antigravity CLI (`agy`) |
|--|-------------|------------|
| Instructions file | `CLAUDE.md` | `AGENTS.md` |
| Skills location | `.claude/skills/` (project-local) | `.agents/skills/` (project-local) |
| Skills mechanism | Relative symlinks → `_vault/vault-*/SKILL.md` | Hard links from `_vault/vault-*/SKILL.md` |
| Skills tracked in git | ❌ gitignored (content lives in `_vault/`) | ❌ gitignored (content lives in `_vault/`) |
| Setup command | `bash _vault/install.sh` — auto-loaded, no reload | `bash _vault/install.sh` then run `agy` |
| Article `llm_model` | `claude-sonnet-4-6`, `claude-opus-4-6`, … | `gemini-3-pro`, `gemini-3-flash`, … |

`install.sh` handles both in one run — Claude symlinks go to `.claude/skills/`, Antigravity hard links land in `.agents/skills/`.

## Why Different Mechanisms

- **Claude Code** uses symlinks so that updating `_vault/vault-*/SKILL.md` is immediately reflected without reinstalling — the symlink just points to the updated file.
- **Antigravity CLI** loads workspace skills from `.agents/skills/`; the installer hard-links them there so a skill edit is reflected without copying. When skills are updated, re-run `bash _vault/install.sh` to refresh the links.

## Migrating from Gemini CLI

Antigravity CLI (`agy`) replaced Gemini CLI. If you have an older vault set up for Gemini CLI:

1. Pull the latest framework (`bash bootstrap.sh update`) or re-run `bash _vault/install.sh`.
2. The installer provisions `.agents/skills/` and removes the legacy `.gemini/skills/` directory.
3. Workspace instructions now live in `AGENTS.md` (renamed from `GEMINI.md`) — Antigravity reads `AGENTS.md` by default.
4. Install the CLI itself: `curl -fsSL https://antigravity.google/cli/install.sh | bash` (a single compiled binary — no Node or Python runtime).

Run `agy inspect` at any time to confirm which context files (`AGENTS.md`) and skills (`.agents/skills/`) are active.

## Switching Between Frontends

You can use Claude Code and Antigravity CLI interchangeably on the same vault — they read the same `wiki/`, `raw/`, and `_vault/` directories. The only differences are:

1. The instructions file loaded (`CLAUDE.md` vs `AGENTS.md`)
2. The `llm_model` recorded in article frontmatter
3. Skill invocation syntax (same skill names; Antigravity also exposes them as slash commands)

## What Stays on GitHub vs. What Stays Local

| | GitHub (the template) | Your local wiki |
|--|----------------------|--------------------|
| `_vault/` skills & engines | ✅ tracked | ✅ tracked |
| `CLAUDE.md`, `AGENTS.md`, `_templates/`, `.obsidian/` | ✅ tracked | ✅ tracked |
| `pyproject.toml` | ✅ tracked | ✅ tracked |
| `.claude/skills/` Claude Code project skills | ❌ gitignored | generated by install.sh |
| `.agents/skills/` Antigravity CLI project skills | ❌ gitignored | generated by install.sh |
| `raw/` source documents | ❌ gitignored | local only |
| `wiki/` compiled articles | ❌ gitignored | local only |
| `viz/` visualizations | ❌ gitignored | local only |
| `outputs/` reports | ❌ gitignored | local only |

Wiki content is deliberately not tracked. Your notes, source documents, and compiled articles can be large, private, or both. Keep them local (or back them up separately — e.g. a private git repo, Obsidian Sync, or a local backup tool).
